Three Smart Questions

  1. In applications where some processing (such as of images) is required, is it ever "okay" to relegate that to the client (via, say, JavaScript), and then upload the results, or is it always better to have the server do it? For example, upon uploading a 2MP image, a smaller thumbnail needs to be generated. Should the server do it or the client computer (assuming the resulting file size is not an issue for the network, maybe only a few KB on top of the 2MP image)?
  2. When a client computer requests a page from the PHP server, we are told that PHP will execute the code and generate a resulting HTML document. Assuming that the resulting HTML is static (and not dynamic/changing frequently), is the resulting HTML page cached by PHP or does it build a new HTML document for every client request?
  3. What are some of the major competitors of PHP? How is PHP contending? Are there any significant benefits of PHP over other languages or vice versa?
